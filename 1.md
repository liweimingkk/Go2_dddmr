DDDMR + Go2 + XT16 + 嘴部雷达地面补点实施文档
0. 目标

当前状态：

上方 XT16 单独跑 DDDMR LeGO-LOAM 已经成功。
嘴部雷达主点云话题：/utlidar/cloud。
上方 XT16 平置时地面点不足，希望利用 Go2 嘴部雷达补地面。

最终目标：

/xt16_points 或当前成功使用的主雷达点云
    -> DDDMR ImageProjection 原流程
    -> projectPointCloud()
    -> zPitchRollFeatureRemoval()
    -> append mouth ground points into patched_ground_
    -> cloudSegmentation()
    -> publishClouds()
    -> FeatureAssociation / MapOptimization

关键原则：

1. 不要把 /utlidar/cloud 全量 concat 到主雷达点云。
2. 不要让嘴部雷达参与 edge/surf 特征提取。
3. 嘴部雷达只提取“接近地面”的 ROI 点。
4. 提取出的嘴部地面点要变换到上方主雷达 frame 后再 push 到 patched_ground_。

原因是 DDDMR 的 ImageProjection 当前只订阅一个 lslidar_point_cloud 输入；它会用单雷达原点计算 verticalAngle、horizonAngle，并把点投到 range image。双雷达全量拼接会破坏这个几何假设。源码中 ImageProjection 订阅 lslidar_point_cloud，projectPointCloud() 用点的 x/y/z 计算行列索引并写入 _range_mat。

1. 已知 TF / 外参关系
1.1 Go2 URDF 中嘴部雷达

Unitree Go2 URDF 里嘴部雷达 link 名字是：

radar

对应 fixed joint：

<joint name="radar_joint" type="fixed">
  <origin xyz="0.28945 0 -0.046825" rpy="0 2.8782 0" />
  <parent link="base" />
  <child link="radar" />
</joint>

也就是：

T_base_radar:
  xyz = 0.28945, 0, -0.046825
  rpy = 0, 2.8782, 0

URDF 同时定义了 base -> imu：

<joint name="imu_joint" type="fixed">
  <origin xyz="-0.02557 0 0.04232" rpy="0 0 0" />
  <parent link="base" />
  <child link="imu" />
</joint>

这两个外参来自 Go2 URDF。

1.2 XT16 与 Go2 内部 IMU

根据你给的图：

T_imu_xt16:
  xyz = 0.171, 0, 0.0908
  rpy = 0, 0, 0

因为 URDF 中：

T_base_imu:
  xyz = -0.02557, 0, 0.04232
  rpy = 0, 0, 0

所以如果当前 TF 树使用 URDF 的 base 作为机身坐标，XT16 可直接写成：

T_base_xt16 = T_base_imu * T_imu_xt16

xyz:
  x = -0.02557 + 0.171   = 0.14543
  y = 0
  z = 0.04232 + 0.0908  = 0.13312

rpy:
  0, 0, 0

也就是：

base -> xt16:
  xyz = 0.14543, 0, 0.13312
  rpy = 0, 0, 0

如果你的运行时 TF 已经有 base -> imu，也可以不要直接发 base -> xt16，而是发：

imu -> xt16:
  xyz = 0.171, 0, 0.0908
  rpy = 0, 0, 0

两种方式二选一，不要同时发布 base -> xt16 和 imu -> xt16，否则 xt16 可能出现重复父节点。

2. 推荐 TF 树

优先使用这个结构：

odom
  └── base_footprint
        └── base
              ├── imu
              │    └── xt16
              ├── radar
              └── front_camera

如果你当前第一步成功的系统已经在使用 base_link、laser_link 或别的 frame 名字，不要强行改名。Codex 只需要保证下面三件事成立：

1. 主 XT16 点云 header.frame_id 能通过 TF 查到 laser.base_ground_frame。
2. /utlidar/cloud 的 header.frame_id 能通过 TF 查到 mouth_filter_frame。
3. mouth_filter_frame 能通过 TF 查到主 XT16 frame。

DDDMR 的 Go2 配置示例中 laser.base_ground_frame 用的是 base_footprint，odom_type 用的是 wheel_odometry，主雷达配置是 16 线。这个方向和你的场景一致，但不要覆盖你已经跑通的主雷达参数。

5.3 在主流程中插入补地面步骤

原流程是：

projectPointCloud();
zPitchRollFeatureRemoval();
cloudSegmentation();
publishClouds();

cloudHandler() 里现在应改为：

// Range image projection
projectPointCloud();

// Mark ground points
zPitchRollFeatureRemoval();

// Append ground-only ROI from Go2 mouth lidar into patched_ground_
appendMouthGroundToPatchedGround(rclcpp::Time(laserCloudMsg->header.stamp));

// Point cloud segmentation
cloudSegmentation();

// publish
publishClouds();

一定要放在 zPitchRollFeatureRemoval() 后面，因为 zPitchRollFeatureRemoval() 会清空并重新生成 patched_ground_ 和 patched_ground_edge_。源码中这个函数开头会 clear 两个 ground cloud，随后再做 ground patch 和 voxel filter。

5.4 实现 appendMouthGroundToPatchedGround()

把下面函数加到 imageProjection.cpp 中。

void ImageProjection::appendMouthGroundToPatchedGround(
    const rclcpp::Time& main_stamp)
{
  if (!enable_mouth_ground_fusion_) {
    return;
  }

  sensor_msgs::msg::PointCloud2::SharedPtr mouth_msg;
  {
    std::lock_guard<std::mutex> lock(mouth_cloud_mutex_);
    mouth_msg = latest_mouth_cloud_;
  }

  if (!mouth_msg) {
    return;
  }

  const double dt =
      std::abs((rclcpp::Time(mouth_msg->header.stamp) - main_stamp).seconds());

  if (mouth_max_time_diff_ > 0.0 && dt > mouth_max_time_diff_) {
    RCLCPP_WARN_THROTTLE(
        this->get_logger(),
        *this->get_clock(),
        2000,
        "Skip mouth cloud: time diff %.3f > %.3f sec",
        dt,
        mouth_max_time_diff_);
    return;
  }

  sensor_msgs::msg::PointCloud2 mouth_in = *mouth_msg;

  if (!mouth_frame_override_.empty()) {
    mouth_in.header.frame_id = mouth_frame_override_;
  }

  if (mouth_in.header.frame_id.empty()) {
    RCLCPP_WARN_THROTTLE(
        this->get_logger(),
        *this->get_clock(),
        2000,
        "Skip mouth cloud: empty frame_id and no mouth_frame_override");
    return;
  }

  const std::string filter_frame =
      mouth_filter_frame_.empty() ? base_ground_frame_ : mouth_filter_frame_;

  geometry_msgs::msg::TransformStamped tf_mouth_to_filter;
  geometry_msgs::msg::TransformStamped tf_filter_to_sensor;

  try {
    tf_mouth_to_filter = tf2Buffer_->lookupTransform(
        filter_frame,
        mouth_in.header.frame_id,
        tf2::TimePointZero);

    tf_filter_to_sensor = tf2Buffer_->lookupTransform(
        sensor_frame_,
        filter_frame,
        tf2::TimePointZero);
  } catch (const tf2::TransformException& ex) {
    RCLCPP_WARN_THROTTLE(
        this->get_logger(),
        *this->get_clock(),
        2000,
        "Skip mouth cloud: TF lookup failed, filter_frame=%s, mouth_frame=%s, sensor_frame=%s, error=%s",
        filter_frame.c_str(),
        mouth_in.header.frame_id.c_str(),
        sensor_frame_.c_str(),
        ex.what());
    return;
  }

  sensor_msgs::msg::PointCloud2 mouth_filter_msg;
  try {
    tf2::doTransform(mouth_in, mouth_filter_msg, tf_mouth_to_filter);
  } catch (const tf2::TransformException& ex) {
    RCLCPP_WARN_THROTTLE(
        this->get_logger(),
        *this->get_clock(),
        2000,
        "Skip mouth cloud: doTransform to filter frame failed: %s",
        ex.what());
    return;
  }

  pcl::PointCloud<PointType>::Ptr mouth_filter_cloud(
      new pcl::PointCloud<PointType>());
  pcl::fromROSMsg(mouth_filter_msg, *mouth_filter_cloud);

  pcl::PointCloud<PointType>::Ptr mouth_ground_filter(
      new pcl::PointCloud<PointType>());

  mouth_ground_filter->reserve(mouth_filter_cloud->size());

  for (const auto& p : mouth_filter_cloud->points) {
    if (!pcl::isFinite(p)) {
      continue;
    }

    const double r = std::sqrt(
        static_cast<double>(p.x) * p.x +
        static_cast<double>(p.y) * p.y +
        static_cast<double>(p.z) * p.z);

    if (r < mouth_range_min_ || r > mouth_range_max_) {
      continue;
    }

    // ROI in filter_frame. For base_footprint this means:
    // x forward, y left/right, z height above ground.
    if (p.x < mouth_x_min_ || p.x > mouth_x_max_) {
      continue;
    }

    if (std::abs(p.y) > mouth_y_abs_) {
      continue;
    }

    if (p.z < mouth_ground_z_min_ || p.z > mouth_ground_z_max_) {
      continue;
    }

    PointType q = p;
    q.intensity = 0.0;
    mouth_ground_filter->push_back(q);
  }

  if (static_cast<int>(mouth_ground_filter->size()) < mouth_min_points_) {
    return;
  }

  Eigen::Affine3d eigen_filter_to_sensor =
      tf2::transformToEigen(tf_filter_to_sensor);

  pcl::PointCloud<PointType>::Ptr mouth_ground_sensor(
      new pcl::PointCloud<PointType>());

  pcl::transformPointCloud(
      *mouth_ground_filter,
      *mouth_ground_sensor,
      eigen_filter_to_sensor);

  if (mouth_voxel_size_ > 0.0) {
    pcl::VoxelGrid<PointType> vg;
    vg.setInputCloud(mouth_ground_sensor);
    vg.setLeafSize(
        static_cast<float>(mouth_voxel_size_),
        static_cast<float>(mouth_voxel_size_),
        static_cast<float>(mouth_voxel_size_));

    pcl::PointCloud<PointType>::Ptr filtered(
        new pcl::PointCloud<PointType>());
    vg.filter(*filtered);
    mouth_ground_sensor = filtered;
  }

  for (auto& p : mouth_ground_sensor->points) {
    if (!pcl::isFinite(p)) {
      continue;
    }

    p.intensity = 0.0;
    patched_ground_->push_back(p);
  }

  if (_pub_mouth_ground_cloud &&
      _pub_mouth_ground_cloud->get_subscription_count() > 0) {
    sensor_msgs::msg::PointCloud2 debug_msg;
    pcl::toROSMsg(*mouth_ground_sensor, debug_msg);
    debug_msg.header = _seg_msg.header;
    debug_msg.header.stamp = _seg_msg.header.stamp;
    debug_msg.header.frame_id = sensor_frame_;
    _pub_mouth_ground_cloud->publish(debug_msg);
  }
}
6. 配置文件修改

复制你第一步已经跑通的配置文件，例如：

cp src/dddmr_lego_loam/lego_loam_bor/config/你的成功配置.yaml \

不要改已经跑通的 XT16 参数。只在 lego_loam_ip.ros__parameters.imageProjection 下面追加：

lego_loam_ip:
  ros__parameters:
    imageProjection:
      enable_mouth_ground_fusion: true

      # Go2 mouth lidar topic
      mouth_cloud_topic: "/utlidar/cloud"

      # 如果 /utlidar/cloud 的 header.frame_id 已经是 radar，可以设为空字符串。
      # 如果 header.frame_id 是空的或不是 TF 树里的 frame，可先强制用 radar。
      mouth_frame_override: "radar"

      # 推荐使用 base_footprint。如果你的第一步配置没有 base_footprint，
      # 可以临时设成 base，但 z 阈值要改成 base 坐标下的地面高度。
      mouth_filter_frame: "base_footprint"

      # 时间同步容忍。两个雷达频率不完全一致时可放宽到 0.12。
      mouth_max_time_diff: 0.08

      # base_footprint 下地面高度范围，单位 m。
      # 地面应在 z≈0 附近。
      mouth_ground_z_min: -0.08
      mouth_ground_z_max: 0.12

      # 只取机器人前方近距离地面，避免嘴部雷达把墙、人、机器人腿加入地面。
      mouth_x_min: 0.05
      mouth_x_max: 3.0
      mouth_y_abs: 1.2

      # 嘴部雷达有效距离范围。
      mouth_range_min: 0.15
      mouth_range_max: 4.5

      # 嘴部地面点单独降采样。
      mouth_voxel_size: 0.06

      # 过滤后点太少则跳过，避免偶发噪声进入 ground map。
      mouth_min_points: 20

如果没有 base_footprint，先临时这样调试：

mouth_filter_frame: "base"

# 下面这两个值要根据 RViz 中 ground 在 base 坐标下的实际 z 来改。
# 例如 Go2 站立时地面可能在 base.z = -0.25~-0.35 附近。
mouth_ground_z_min: -0.42
mouth_ground_z_max: -0.18

但长期建议还是提供 base_footprint，因为 DDDMR 的 Go2 示例配置本身也是以 base_footprint 作为 laser.base_ground_frame。

7. Launch 修改建议

DDDMR 原 launch 里默认加载 loam_c16_config.yaml，并把 /lslidar_point_cloud remap 到自己，还发布了一个 base_link -> laser_link 的静态 TF。你这里不要继续用那个默认静态 TF，要换成 Go2 + XT16 + radar 的 TF。

建议新建：

src/dddmr_lego_loam/lego_loam_bor/launch/lego_loam_go2_xt16_mouth.launch

模板如下。把 /YOUR_XT16_TOPIC 换成你第一步已经跑通的 XT16 主点云话题。

<launch>
  <node pkg="lego_loam_bor" exec="lego_loam" output="screen" respawn="false">
    <param from="$(find-pkg-share lego_loam_bor)/config/loam_go2_xt16_mouth_config.yaml" />

    <!-- 主雷达输入：保持你第一步已成功的话题 -->
    <remap from="/lslidar_point_cloud" to="/YOUR_XT16_TOPIC"/>

    <!-- Go2/外部里程计 -->
    <remap from="/odom" to="/odom"/>
  </node>

  <!-- 可选：RViz -->
  <node pkg="rviz2" exec="rviz2" output="screen" respawn="false"
        args="-d $(find-pkg-share lego_loam_bor)/rviz/lego_loam.rviz" />

  <!-- TF：优先用 robot_state_publisher 发布 URDF。
       如果 robot_state_publisher 已经在发 base->imu、base->radar，不要重复发布下面这些。 -->

  <!-- 方式 A：如果已有 base->imu，则只发 imu->xt16 -->
  <!--
  <node pkg="tf2_ros" exec="static_transform_publisher"
        name="imu_to_xt16"
        args="0.171 0 0.0908 0 0 0 imu xt16" />
  -->

  <!-- 方式 B：如果不用 imu 链，直接发 base->xt16 -->
  <!--
  <node pkg="tf2_ros" exec="static_transform_publisher"
        name="base_to_xt16"
        args="0.14543 0 0.13312 0 0 0 base xt16" />
  -->

  <!-- 如果没有 robot_state_publisher 发布 Go2 URDF，则至少需要 base->radar -->
  <!--
  <node pkg="tf2_ros" exec="static_transform_publisher"
        name="base_to_radar"
        args="0.28945 0 -0.046825 0 2.8782 0 base radar" />
  -->

  <!-- 如果 /utlidar/cloud 的 frame_id 是 utlidar_lidar，但实际坐标就是 radar，

8. 运行前检查
8.1 检查嘴部点云 frame
ros2 topic echo /utlidar/cloud --once | grep frame_id

常见情况：

frame_id: radar

这种情况：

mouth_frame_override: ""

如果输出：

frame_id: utlidar_lidar

有两种做法：

做法 1：发布 radar -> utlidar_lidar 的 identity TF。
做法 2：如果确定 /utlidar/cloud 坐标实际就是 radar，则 mouth_frame_override: "radar"。
8.2 检查 TF

至少要能查通：

ros2 run tf2_ros tf2_echo base radar
ros2 run tf2_ros tf2_echo base xt16
ros2 run tf2_ros tf2_echo xt16 radar

如果使用 base_footprint，还要查：

ros2 run tf2_ros tf2_echo base_footprint xt16
ros2 run tf2_ros tf2_echo base_footprint radar

如果这里失败，DDDMR 里也会失败。源码里 ImageProjection::allEssentialTFReady() 会查 base_ground_frame -> sensor_frame，查不到就报 footprint 到 sensor frame 的 TF 错误。

8.3 检查话题频率
ros2 topic hz /utlidar/cloud
ros2 topic hz /YOUR_XT16_TOPIC

两者时间差太大时，把：

mouth_max_time_diff: 0.08

临时放宽到：

mouth_max_time_diff: 0.12

11. 验收标准

Codex 完成后，应满足：

1. enable_mouth_ground_fusion=false 时，DDDMR 行为与原版一致。
2. enable_mouth_ground_fusion=true 时，节点订阅 /utlidar/cloud。
3. /mouth_ground_cloud 能发布，frame_id 等于主 XT16 frame。
4. /ground_cloud 中包含嘴部雷达补充地面点。
5. /segmented_cloud_pure 和特征提取不直接使用 /utlidar/cloud 的非地面点。
6. 不修改 FeatureAssociation / MapOptimization。
7. 不破坏原来单 XT16 建图能力。
12. 最容易踩坑的地方

第一，/utlidar/cloud 的 frame_id 不一定叫 radar。先 ros2 topic echo 看清楚，再决定用 mouth_frame_override 还是补 identity TF。

第二，不要重复发布 TF。尤其是 base -> radar，如果 robot_state_publisher 已经发了，就不要再用 static_transform_publisher 发一次。

第三，base_footprint 如果不存在，不要直接把 mouth_ground_z_min/max 设成 -0.08~0.12。这个范围只适用于地面在 z≈0 的 frame。如果用 base 过滤，z 范围必须围绕 “地面在 base 坐标下的 z 值”。

第四，嘴部雷达补点只进 patched_ground_，不要进 _laser_cloud_in。_laser_cloud_in 会被投影到 range image，双雷达原点不同，会制造错误行列索引。

第五，XT16 的静态外参按你图和 URDF 合成后是：

base -> xt16:
  xyz = 0.14543, 0, 0.13312
  rpy = 0, 0, 0
